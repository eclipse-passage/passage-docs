Passage puts you product and/or it's features under license protection.

To make it do so you should take several steps.

  • Done by product development:

    • declare what in the product is protected by license (it can be the whole product or/and some particular features);
    • implant license checks in proper points in the product codebase;
    • configure Passage Access Cycle for your particular needs or clone one of the existing configurations.

    From this point the product is treated as protected by Passage.

    Now we need to issue a license for our product to make Passage let ua user to use the product. This is done with two steps.

  • Done by product management: create licensing metadata for the product

    • define product, features and their versions under licensing protection,
    • generate encryption keys,
    • create license plan

    These step begets several artifacts (now xmi files) that are quite precious and should be kept with care.

  • Done by sales management:

    license issuing boils down to a simple wizard passing which ends up by a pack of license files storing on a file system.

  • Done by doc team:

    This step is to be performed by the product's user, but should definitely be documented: when a user gets your product and a license, how they should apply the license to the product.
